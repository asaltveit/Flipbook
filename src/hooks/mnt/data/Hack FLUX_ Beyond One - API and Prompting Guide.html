Foundational FLUX.2 prompting framework

Based on that prompt usually build around the following framework:
Subject + Action + Style + Context
Framework Explanation:
Subject: The main focus (person, object, character)
Action: What the subject is doing or their pose
Style: Artistic approach, medium, or aesthetic
Context: Setting, lighting, time, mood, or atmospheric conditions

Negative Prompting in FLUX.2

FLUX.2 does not support negative prompts. If you use negative prompts, it might have the opposite effects and adds what you don‚Äôt want to. You should rely on positive constraints:
Be specific: describe what you do want (subject, action, style, context).
Use structured JSON for precise control (pose, composition, camera).

FLUX.2 Image-To-Image Prompting

Core Capabilities

Multi-reference editing - Use multiple images in a single edit.
Sequential Edits - Edit images sequentially.
Color control with hex codes - Specify exact colors using hex values (e.g., #B76E79) and an input image for precise color matching
Explicit image indexing - Reference specific images by number: "the jacket from image 2" or "replace the background with image 3."
Natural language - Describe elements from your input images and FLUX.2 understands them: "the woman in the blue dress" or "the burgundy jacket."
Collage inputs - Use a layout collage as a single input image to guide composition. Lower quality than multi-reference, but useful for quick layout experiments.

Using reference inputs

FLUX.2 supports using multiple images as an input (up to 6 images). You can pass several images at once and tell the model which attribute to use from which image.
How to write it:
Name the attribute and point to the source image by index number.
Describe the subject you want to use from the different images
